国赛向量索引优化赛题中遇到的问题：

## SQ4 和 SQ8 的实现

**Q: SQ4 和 SQ8 的实现？**

答：

- **SQ4**：将 0-128 的值域映射至 0-16，损失了一部分精度（有损映射），但减少了数据量，提高了带宽和运算速度。QPS 从 1400 提升到 1600，但召回率下降到约 70%。

- **SQ8**：将 0-255 的值域映射至 0-255，精度损失较小，召回率更高，但数据量更大，带宽占用更高。

优化方案：在 SQ8 的基础上进行重排（re-ranking），平衡召回率和性能。

## 算法题

### 判断链表是否有环

写了 Floyd 判圈算法（快慢指针），AC 了。

```cpp
bool hasCycle(ListNode *head) {
    if (!head || !head->next) return false;

    ListNode *slow = head;
    ListNode *fast = head->next;

    while (fast && fast->next) {
        if (slow == fast) return true;
        slow = slow->next;
        fast = fast->next->next;
    }

    return false;
}
```

## 一致性问题

**Q: 强一致性、弱一致性、最终一致性、线性一致性？**

这个问题是面试的重点，考察对分布式系统一致性的理解。

### 强一致性（Strong Consistency）

系统在执行数据更新操作后，所有的读操作都能立即读取到最新的数据。

- 特点：读写操作是同步的，更新操作会阻塞后续读操作，直到数据同步完成
- 场景：金融系统、数据库事务

### 弱一致性（Weak Consistency）

系统不保证后续访问能否读取到最新写入的数据，只保证在某一时刻之后能够读取到。

- 特点：不保证读操作能立即读到最新数据
- 场景：缓存系统、CDN

### 最终一致性（Eventual Consistency）

弱一致性的一种特例，系统保证在没有新的更新操作后，最终所有访问都能读取到最新的数据。

- 特点：有短暂的不一致窗口，但最终会收敛
- 场景：DNS 系统、分布式数据库（如 Dynamo、Cassandra）

### 线性一致性（Linearizability）

最强的一致性模型，要求操作看起来是原子执行的，且执行顺序与全局时钟顺序一致。

- 特点：不仅要求强一致性，还要求操作的全局顺序性
- 场景：分布式锁、协调服务（如 etcd、ZooKeeper）

### 反思

这个问题回答得不够系统，应该先下定义，再对比区别，最后举例说明。尤其线性一致性和强一致性的区别没有说清楚——线性一致性额外要求操作的全局时钟顺序。
