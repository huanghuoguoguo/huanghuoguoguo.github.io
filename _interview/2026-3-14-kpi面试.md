## JavaCPP

自我介绍完毕，对我简历上感兴趣的地方只有 JavaCPP。问了实习时做的相关项目。

**Q: JavaCPP 是怎么实现的，和 JNI 有什么区别？**

> JavaCPP 本质上是个 JNI 的代码生成器。你用注解标注 Java 类，它自动生成 JNI 胶水代码（`.cpp` 文件），再编译成 `.so`/`.dll`。和手写 JNI 的区别主要是省去了写 C/C++ 桥接层的工作，映射关系靠注解声明式完成。

**Q: 那你知道 JNI 是怎么实现的吗？**

我说就是绑定一下吧。

> JNI 通过 `System.loadLibrary` 加载动态链接库，JVM 根据方法签名找到 native 方法对应的 C/C++ 函数。核心是 `JNIEnv` 指针，提供了一张函数表，Java 和 native 代码通过这张表互相调用。

## Java 八股

**Q: Java 线程间通信有哪些方式？**

我提到了 `synchronized` 和 JUC 下的 `Lock`。其实后面还问到了什么时候使用什么锁方式。

> 共享内存：`volatile`、`synchronized`、`Lock`；等待通知：`wait()`/`notify()`、`Condition`、`LockSupport.park()`/`unpark()`；工具类：`CountDownLatch`、`CyclicBarrier`、`Semaphore`；队列：`BlockingQueue`；管道：`PipedInputStream`/`PipedOutputStream`。

**Q: synchronized 和 Lock 有什么区别？**

> `synchronized` 是 JVM 内置的，基于 Monitor 对象实现，自动释放锁；`Lock` 是 API 层面的，基于 AQS 实现，需要手动 `unlock()`，支持可中断、超时、公平锁等更灵活的控制。

接下来聊到了 JUC 的实现方式——AQS、LockSupport、CAS。

**Q: CAS 你知道是怎么实现的吗？**

> Java 的 CAS 通过 `Unsafe` 类调用，底层对应 CPU 的 `cmpxchg` 指令。单核不需要额外保证；多核下需要加 `lock` 前缀，锁缓存行或总线来保证原子性。

**Q: 抽象类和接口的区别？**

一系列问题，不展开了。

> 抽象类可以有构造方法、成员变量、具体方法实现，单继承；接口只能有常量和抽象方法（Java 8+ 可以有 default 和 static 方法），多实现。语义上抽象类是 "is-a"，接口是 "can-do"。

**Q: 死锁条件？怎么判别和解除死锁？**

答了三个条件，忘了。

> 四个必要条件：互斥、占有并等待、不可抢占、循环等待。判别可以用资源分配图检测环路，或者 `jstack` 直接看线程 dump。解除的话，杀进程或者剥夺资源。

**Q: wait、notify、sleep 的区别？**

我都聊到 JVM Object Monitor，等待队列、通知唤醒机制，和 AQS 的区别了。

> `wait()` 是 Object 的方法，必须在 synchronized 块里调用，会释放锁，线程进入等待队列，被 `notify()`/`notifyAll()` 唤醒后重新竞争锁。`sleep()` 是 Thread 的静态方法，不释放锁，到时间自动恢复。`notify()` 随机唤醒一个等待线程，`notifyAll()` 唤醒所有。

**Q: 泛型。**

> Java 泛型是编译期的类型检查，运行时会类型擦除，泛型信息不保留（`List<String>` 和 `List<Integer>` 运行时都是 `List`）。通配符：`<? extends T>` 上界只读，`<? super T>` 下界只写（PECS 原则：Producer Extends, Consumer Super）。

**Q: 设计一个 HashMap，怎么解决 hash 冲突？**

> 拉链法或开放寻址。Java 的 HashMap 用拉链法，数组 + 链表，链表长度超过 8 且数组长度超过 64 时转红黑树。

**Q: 数组和链表的区别？**

> 数组连续内存，随机访问 O(1)，插入删除 O(n)，缓存友好；链表非连续内存，随机访问 O(n)，插入删除 O(1)（已知位置的话），每个节点额外存指针。

**Q: 数据库 ACID？**

> Atomicity 原子性：事务要么全做要么全不做，靠 undo log 回滚。Consistency 一致性：事务前后数据满足约束。Isolation 隔离性：并发事务互不干扰，靠锁和 MVCC 实现，四个隔离级别（读未提交、读已提交、可重复读、串行化）。Durability 持久性：提交后数据不丢，靠 redo log + WAL。

**Q: 数据库事务，MVCC，两个事务同时更新一条记录会发生什么？**

> MVCC 在读的时候通过版本链避免加锁，但更新的时候还是要加行锁的。两个事务同时 UPDATE 同一行，先拿到锁的执行，后到的阻塞等待。如果等待超时或检测到死锁，回滚其中一个。
