最近在续写一本小说，167 章，一开始纯用 Claude Code 来写。

具体怎么写的？建一个文件夹，扔一堆 md 文件进去——章节、角色设定、大纲、世界观、叙事节奏、伏笔清单、风格指南。然后让 agent 自己去读、自己改、自己维护。read、write、update，全交给它。

一时很爽。迭代起来要了命。

多份角色设定散在不同目录，多份大纲版本并存，多份任务窗口开着。你根本不知道 agent 当前在读哪个版本、改的是哪个文件。下次开新会话，它又从头扫一遍目录，挑它觉得"相关"的内容塞进上下文。这次挑的是 `characters.md`，下次可能挑的是 `plot_index.md` 里某一行提到的人物设定。两份文件写的东西不一样，agent 自己不知道，你也不知道，直到写出来的章节里人物性格又漂了。

老实说，这时候写的还行。就是经常犯蠢——上下文接不上、前后矛盾、风格提取不对。我试了一圈：调 prompt、加记忆、分章节写、写完一轮再审一轮。有改善，但每次开新会话又回到原点。

后来决定找找开源。想看看有没有现成的写作平台，把世界观管理、RAG、结构化存储这些事做好。

试了两个。都不太行。

一个叫 Author，AI 辅助写作平台，README 上功能列了一长串——Tiptap 编辑器、多模型支持、世界观管理、RAG 向量化。Fork 过来，commit 了 61 次，改到一半卡住了。

卡住的原因倒不是功能不够。是它的关注点太奇怪。

核心功能——AI 辅助写作——做得一塌糊涂。AI 助手只能往文档里插入文字，不能按需编辑、不能改写、不能选中一段让它润色。你要改中间某句话，只能自己动手。那这 AI 助手有什么用？

旁路功能倒是堆了一堆：SaaS 部署、域名反代、账号管理、云端同步、多语言支持。README 维护到了蒙古语。我实在无法理解。核心功能不好用，旁路倒是搞得挺热闹。

然后是这个文件。

---

`SettingsPanel.js`。

3270 行，225KB。一个 React 组件。一个文件。

![author-bad](/images/author-bad.png)

你感受一下。我翻了几屏就想骂人了。满屏幕的 `style={{}}`，hover 效果用 `onMouseEnter` 加 `onMouseLeave` 手动切 style 对象。import 冲突检测的逻辑在 JSON、PMPX、TXT 三个路径里原样重抄了一遍，15 个 `useState` 互相依赖，`console.log` 散了一地。行内样式铺到每一行 JSX 上，读起来像一份 CSS 被塞进了 JavaScript 里再被搅拌机打过一遍。

这他妈的没法维护。

---

我不是说原作者不行。96 个 star，21 个 release，功能是跑起来的。但你一眼就能看出来，这代码是 AI 生成的。

Claude Code 帮你写一个设置面板，它会很勤快地一行行给你补。你问它"能不能把这个组件拆成几个文件"，它也很勤快地帮你拆。但第一次生成的时候，你不会要求它拆——因为你在那个时候还不知道一个设置面板会长到 3000 行。你只说了"我要一个设置面板"，然后它就开始堆。

AI 不管以后。它只管你当前这句 prompt。你不说拆，它就全塞一个文件里。你不说抽公共逻辑，它复制粘贴三遍。你不说加 CSS modules，它就给你写行内样式——行内样式最简单，不用额外上下文，堆就完了。

**生成的时候很爽。维护的时候你是个孙子。**

---

我 fork 过来之后也是 vibe 着改——Claude Code 改这里、改那里，61 个 commit 过去了。越改越发现不对：我在帮 AI 产出的代码擦屁股。它能跑，但不能维护。它能用，但读不下去。

走到这一步我停了。没有继续改，没有让 AI 重构。

我把这个项目扔了。

61 个 commit，当沉默成本处理掉了。之前那些改动确实都是尝试——试了一下才发现这路走不通。再继续投入就是给一个错误的开头买单，没意义。

---

我换了个方向。

回到最开始那个问题：用 Claude Code 直接改小说 md 文件，漂移。人物性格漂、文风漂、上下文接不上。

之前想的是找一个现成的写作平台，有世界观管理、有 RAG、有结构化存储。找到 Author，fork 过来改，改到 SettingsPanel.js 卡住。

现在想的是：不找平台了。我自己搭一个 CLI。

思路很简单——不让 AI 直接碰 md 文件。给它一个程序化的入口：CLI 命令、明确的参数、结构化的输入输出。它只能通过这个入口来改文档，改什么、改哪里、怎么改，都由 CLI 约束。这样漂移的问题就在程序层面被卡住了，而不是靠 AI 自己"记得"不要乱改。

---

不懂 coding 的人，遇到 SettingsPanel.js 这种东西，只能硬着头皮继续让 AI 改，改出更多问题，再让 AI 改，死循环。

我运气好点。能判断这代码值不值得继续投入，能判断什么时候该扔掉、什么时候该换路，能自己搭一个 CLI 把 AI 的行为框住。
