## JavaCPP

自我介绍完毕，对我简历上感兴趣的地方只有 JavaCPP。问了实习时做的相关项目。

**Q: JavaCPP 是怎么实现的，和 JNI 有什么区别？**

> JavaCPP 本质上是个 JNI 的代码生成器。你用注解标注 Java 类，它自动生成 JNI 胶水代码（`.cpp` 文件），再编译成 `.so`/`.dll`。和手写 JNI 的区别主要是省去了写 C/C++ 桥接层的工作，映射关系靠注解声明式完成。

**Q: 那你知道 JNI 是怎么实现的吗？**

我说就是绑定一下吧。

> JNI 通过 `System.loadLibrary` 加载动态链接库，JVM 根据方法签名找到 native 方法对应的 C/C++ 函数。核心是 `JNIEnv` 指针，提供了一张函数表，Java 和 native 代码通过这张表互相调用。

**Q: 你们项目是链接的什么？JavaCPP 里链接一个 C++ 类怎么写的？**

## Java 八股

**Q: Java 线程间通信有哪些方式？**

我提到了 `synchronized` 和 JUC 下的 `Lock`。其实后面还问到了什么时候使用什么锁方式。

**Q: synchronized 和 Lock 有什么区别？**

> `synchronized` 是 JVM 内置的，基于 Monitor 对象实现，自动释放锁；`Lock` 是 API 层面的，基于 AQS 实现，需要手动 `unlock()`，支持可中断、超时、公平锁等更灵活的控制。

接下来聊到了 JUC 的实现方式——AQS、LockSupport、CAS。

**Q: CAS 你知道是怎么实现的吗？**

> Java 的 CAS 通过 `Unsafe` 类调用，底层对应 CPU 的 `cmpxchg` 指令。单核不需要额外保证；多核下需要加 `lock` 前缀，锁缓存行或总线来保证原子性。
