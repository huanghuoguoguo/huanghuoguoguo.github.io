本文主要介绍标量量化（Scalar Quantization, SQ）的原理，包括训练（Training）、编码（Encoding）与解码（Decoding）过程，并结合 OceanBase 向量索引优化阶段的具体应用场景，探讨 SQ8 和 SQ4 在提升系统带宽与查询速度方面的表现。

## 背景

在 OceanBase 的向量索引优化阶段，我们引入了标量量化技术。其核心目标是：将原始的 `float32` 向量量化为 `int8` 或更低位数的格式，在保证精度可接受的前提下，尽可能提高系统带宽，降低存储开销，并提升检索速度。

简单来说：是否存在一种方法，能将 $d$ 维 `float` 向量编码为 $d$ 维 `int8` 向量，且解码后与原始向量的误差尽可能小？这样存储空间可降低为原来的 1/4，且读取性能大幅提升。答案是肯定的，这就是 **Scalar Quantization (SQ)**。

---

## Scalar Quantization 原理

标量量化通常分为三个核心过程：

1.  **Training (训练)**：确定编码所需的参数（各维度的极大值与极小值）。
2.  **Encode (编码)**：将 `float` 向量量化为 `int8`（或其他位数，如 4-bit）。
3.  **Decode (解码)**：将量化后的数据还原为 `float` 向量以便计算。

为了更好地说明原理，下图展示了标量量化的原理框图：

![Scalar Quantization 原理框图](/images/sq_principle.png)

### 1. Scalar Quantization 训练

训练过程主要是从样本中随机采样 $N$ 个数据，计算并记录每 1 维的最大值（$max_i$）与最小值（$min_i$）。在实际工程中，$N$ 越大，量化边界通常越准确。

### 2. Scalar Quantization 编码

对于 $d$ 维待编码向量 $x = \{x_1, x_2, \dots, x_d\}$：
-   对每一维求：$value_i = \frac{x_i - min_i}{max_i - min_i}$。
-   异常处理：若 $value_i < 0$ 则置为 0；若 $value_i > 1$ 则置为 1。
-   计算编码：$code_i = \text{int}(255 \times value_i)$（针对 8-bit 量化）。

### 3. Scalar Quantization 解码

解码是编码的逆过程：
-   $x_i = min_i + \frac{(code_i + 0.5) \times (max_i - min_i)}{255}$。
-   **Q: 为什么需要加 0.5？** 这是为了取量化区间的中间值，从而减少量化误差。

## Scalar Quantization 实现

以下是基于 Faiss 风格或自定义实现的 SQ 训练示例：

```cpp
// 使用 Faiss 进行 SQ8 训练
faiss::IndexScalarQuantizer SQuantizer(d, faiss::ScalarQuantizer::QT_8bit, faiss::METRIC_L2);
SQuantizer.train(num_db, xb);
// SQuantizer.add(num_db, xb);    
faiss::write_index(&SQuantizer, model_path.c_str());
```

### Encode 实现参考

```cpp
void encode_vector(const float* x, uint8_t* code) const final {
    for (size_t i = 0; i < d; i++) {
        float xi = 0;
        if (vdiff != 0) {
            xi = (x[i] - vmin) / vdiff;
            if (xi < 0) xi = 0;
            if (xi > 1.0) xi = 1.0;
        }
        Codec::encode_component(xi, code, i); // 将 0.0~1.0 映射到 0~255
    }
}
```

### Decode 实现参考

```cpp
void decode_vector(const uint8_t* code, float* x) const final {
    for (size_t i = 0; i < d; i++) {
        float xi = Codec::decode_component(code, i);
        x[i] = vmin + xi * vdiff;
    }
}
```

> **注意**：Faiss 的解码接口在小批量数据下，由于多线程开销，性能可能不升反降。

## 在 OceanBase 中的应用

在 HNSW 索引中，我们可以预先计算每个向量的 SQ4/SQ8 编码，待检索计算时直接使用。

### SIFT 数据集适配性

针对 `sift128-1m` 数据集（值域 [0,128]），由于其天然的分布特征，非常适合使用 `uint8` 存储。

![SIFT 数据集特点](/images/sift_dataset_features.png)

-   **带宽优化**：从 128 维 `float` (512 字节) 降低到 `uint8` (128 字节)，存储降低 4 倍。
-   **指令集加速**：提高系统与缓存带宽利用率。

### SQ8 优化尝试

对于 SIFT 数据集，虽然是 0-128（4-bit 范围），我们先尝试 SQ8 量化。
-   **归一化考量**：大部分数据是同分布的，归一化会破坏 `int8` 的数值关系，因此**不进行归一化**。
-   **性能表现**：使用朴素算法的 SQ8 时，相比原始参数，QPS 达到 **830**。

![SQ8 QPS 830 表现](/images/sq8_qps_830.png)

## 高性能计算优化：AVX512 实践

为了追求极致性能，我们尝试使用 AVX512 指令集处理向量间 L2 距离的计算。

### 计算策略
1.  一次性加载 64 个 `int8` 元素到 512 位寄存器。
2.  计算差值（`diff`）。
3.  **挑战**：`int8` 的平方会超过 `int8` 范围，需升维至 `int16`/`int32` 存储。

### 踩坑总结
直接在 512 位寄存器里做 `int8` 的平方操作会发生溢出或覆盖。常见的成熟方案是将数据偏移缩放回 `float`，但这会丢失 `int8` 的带宽优势。我们希望尽可能在 `int` 域内完成运算。

#### 优化版 SQ8 L2 计算 (1300 QPS)

通过将 `int8` 转换为 `int32` 进行累加，我们获得了约 **1300** 的 QPS 提升。

![SQ8 QPS 1300 表现](/images/sq8_qps_1300_performance.png)

```cpp
float SQ8ComputeCodesL2Sqr(const void* pVect1v, const void* pVect2v, const void* qty_ptr) {
    uint8_t* x = (uint8_t*)pVect1v;
    uint8_t* y = (uint8_t*)pVect2v;

    __m512i sum = _mm512_setzero_si512(); 
    for (int i = 0; i + 15 < 128; i += 16) { 
        __m128i code1 = _mm_loadu_si128((const __m128i*)(x + i)); 
        __m128i code2 = _mm_loadu_si128((const __m128i*)(y + i)); 

        __m512i c1_512 = _mm512_cvtepu8_epi32(code1); 
        __m512i c2_512 = _mm512_cvtepu8_epi32(code2); 

        __m512i diff = _mm512_sub_epi32(c1_512, c2_512); 
        __m512i diff_sq = _mm512_mullo_epi32(diff, diff); 

        sum = _mm512_add_epi32(sum, diff_sq); 
    } 

    int result[16];
    _mm512_storeu_si512(result, sum);
    int total = 0;
    for (int i = 0; i < 16; ++i) total += result[i];

    return (float)total; 
}
```

## SQ4 进一步优化

SQ4 将存储空间从 512 字节进一步降低到 64 字节（恰好一个 Cache Line 大小）。

**优化逻辑：**
-   **粗算 (Coarse)**：在 HNSW 搜索过程中，使用 SQ4 计算近似距离。当两个候选点的距离差距足够小时，再进行精算。
-   **精算 (Fine)**：使用 SQ8 或原始向量进行精确重新排序（Re-ranking）。
-   **重排 (Re-rank)**：确保召回率不因量化损失而下降。

**结果**：使用 SQ4 后，QPS 从 1400 进一步提升至 **1600**。
