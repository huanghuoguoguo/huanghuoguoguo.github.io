# 作者画像维护

这个 skill 的核心是**认知框架**——抽象出用户看待世界、看待知识、看待表达的底层假设。

具体经历、判断、偏好都是认知框架的证据和表现，不是孤立记录。

## 与其他 skill 的联动

### 写文章时（new-or-review）

先读认知框架 → 决定文章推进方式（展示认知过程、允许留白、展示 trade-off）→ 再调用具体经历做证据

### 改写时（humanizer-zh）

先读表达观 → 理解为什么不喜欢某些表达 → 决定改写方向（删掉假装全知的句子、保留带锚点的语气）

---

## 核心文件

- `PROFILE.md`
  - 认知框架（顶层）：知识观、表达观、工程观、成长观
  - 经历：作为认知框架的证据来源，标注认知转折
  - 技术判断：每条标注来路
  - 思考方式/风格偏好：每条标注认知原因

- `references/extraction-and-writeback.md`
  - 抽取流程：优先抽象认知维度，追问”为什么”
  - 落点规则：每条信息关联认知框架

---

## 使用顺序

1. 先读 `PROFILE.md` 认知框架部分
2. 判断本轮有没有新信息
3. 对每条新信息追问认知来路（关联哪个认知维度）
4. 决定落点
5. 用最小改动更新
6. 如有更新，运行检查脚本

---

## 核心原则

1. **优先抽象认知维度**：记录”为什么”，不是只记录”是什么”
2. **每条信息关联认知框架**：判断、表现都要标注认知来路
3. **经历是证据**：每段经历要标注认知转折，不是孤立事实
4. **只记事实和一手判断**：不记没有把握的推测
