# 审稿与修订

## Review 的三部分

首稿完成后，`review` 由三部分组成：

1. 技术事实检查
2. 作者风格检查
3. 调用检测器做结构化分析

## 技术事实检查

至少检查：

- 术语、原理、数据是否正确
- 前提是否支撑结论
- 没有数据支撑的地方是否写得过满
- 有没有超出作者知识边界的展开
- `frontmatter` 是否完整

## 作者风格检查

至少检查：

- 开头是不是具体问题或场景
- 有没有 benchmark、命令输出、syscall 计数、日志、ASCII 图、对比表
- 是否写出了 `V1 -> V2 -> V3` 的真实演化
- 句子是否过长、过于整齐
- 有没有允许闲笔和真实犹豫
- 结尾有没有强行升华
- 判断是不是从经历和观察里长出来的，而不是像讲师一样直接宣布“本质”
- 有没有“先有结论，再回头组织文章”的痕迹
- 有没有把原本带情绪的判断磨成过于中性、过于稳妥的审校腔
- 文中有没有至少几处只能由作者本人写出来的材料或语气
- 有没有哪一段一眼看去就像“这里其实应该先访谈，再写”
- 如果这篇借了样本文风，检查借来的是骨架还是人格：
  - 骨架可以借
  - 样本作者的职业身份、强行教学口吻、固定口头禅不能硬套

## 调用检测器

检测器是外部审查工具，路径：`tools/ai-writing-bad-cases-app/`（相对当前 skill 目录）

如果该路径不存在：

- 明确报告检测器缺失
- 不要声称已经完成检测器审查
- 只能继续做人工 review，并把“缺少检测器结果”列为残余风险

预期使用方式：

- 把当前文章文本交给检测器
- 读取 JSON 输出
- 重点关注段落级 `signals`
- 把命中项转成 revision plan

如果检测器返回以下类型信号，按下述方向理解：

- `cliche_phrase`
  - 套话或空洞过渡过多

- `forced_summary`
  - 段落或小节结尾过度收束

- `uniform_sentence_length`
  - 节奏太平，句长太匀

- `perfect_ladder`
  - 叙事过于工整，不像真实迭代

- `missing_evidence`
  - 该有证据的地方没落地

- `author_fit_low`
  - 整体语气、选材或结构不像作者

- `need_interview`
  - 当前问题不是润色能解决的，应该回访作者补材料

## Review 输出

按严重程度排序输出：

- ❌ 事实错误 / 严重 AI 模板 / 严重风格偏差
- ⚠️ 逻辑问题 / 明显 AI 痕迹 / 证据不足
- 💡 风格优化建议 / 节奏优化建议

每条意见至少包含：

1. 具体位置
2. 问题类型
3. 为什么是问题
4. 修改方向
5. 必要时给出局部改写示例

最后补三项总结：

- 当前 AI 痕迹程度：轻 / 中 / 重
- 最优先处理的 2 到 3 个问题
- 文中最有人味、最该保留的段落和原因

## Revise 原则

修订只针对命中问题做局部修改。

禁止：

- 整篇重写
- 把真实细节删掉换成平滑空话
- 为了过检测把语气改得过于油滑
- 明明应该回访作者，却擅自用常识补完整段

## 按信号类型修订

### `cliche_phrase`

- 删除套话
- 直接进入具体观察、事实或动作

### `forced_summary`

- 删除节尾抽象总结
- 让段落停在具体细节、现象或疑问处

### `uniform_sentence_length`

- 拆长句
- 引入短句和停顿
- 不要求每段都匀称好看

### `perfect_ladder`

- 恢复真实迭代过程
- 写出回退、犹豫、瓶颈迁移和未解决的问题

### `missing_evidence`

- 补命令、日志、数据、图、对比表
- 没证据就收窄结论，不强写

### `author_fit_low`

- 回到作者画像
- 收窄话题范围
- 换成更口语、更朴素、更像项目复盘的叙述方式
- 把“定义式判断”改成“认知来路 + 我的理解”
- 如果问题来自“像样本作者，不像作者本人”，保留结构推进，替换成作者自己的材料、迟疑和判断

### `need_interview`

- 停止继续润色
- 回到作者那里补场景、补认知来路、补边界
- 只在拿到作者回答后继续写

## Final Check

修订后再审一次，决定：

- `accept`
- `retry`
- `handoff_to_human`

停止条件满足任一条就不要继续自动回环：

- 高严重度问题已经清空
- 连续两轮改动后收益很小
- 修订开始损伤真实细节
- 文章已经“没 AI 味，但也没你味”
