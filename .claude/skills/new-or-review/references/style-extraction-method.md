# 如何提取文风

这份方法不是教你复制口头禅，而是把样本文字拆成可观察、可训练、可迁移的几层。

## 一、先分层，不要一上来学句子

提取文风时，先按下面六层拆：

1. 入口层
   - 文章通常从什么东西切进去：问题、场景、人物、情绪、冲突、数据

2. 结构层
   - 作者怎么推进：先讲误解，再讲修正；先给案例，再抽象；还是先抛判断，再回填材料

3. 证据层
   - 作者靠什么让人信：实验、日志、类比、故事、对话、图、代码、历史经历

4. 句法层
   - 句子长短、停顿、口语程度、反问频率、是否爱用并列句

5. 立场层
   - 作者是讲课、聊天、争辩、复盘，还是边写边想

6. 收尾层
   - 如何结束：抽象总结、回到个人感受、给建议、留下疑问，还是说完就停

如果只学第 4 层，最容易写成“像，但假”。真正像，靠前 3 层和第 5 层。

## 二、做两份清单

读样本时，把观察分成两份：

### A. 高置信度的可观察特征

只能写那些你真的在文本里反复看见的东西。

例如：

- 经常从一个具体场景开头
- 经常先写早期误解
- 经常用具体例子推动抽象判断

### B. 低置信度的推测性偏好

这类结论只写“可能”，不要写成定论。

例如：

- 可能偏好先建立读者情绪，再推进技术解释
- 可能不喜欢太学术化的定义口吻

## 三、每条特征都要落成动作

不要只写“风格比较口语”“有感染力”。这种描述没法训练。

每条特征后面都补一个动作：

- 观察：开头常从人和场景切入
- 动作：写提纲时，先补“谁、什么时候、为什么开始在意这个问题”

- 观察：喜欢先写误解再写修正
- 动作：每个大节先问自己“我最早想错了什么”

- 观察：结尾不爱空泛升华
- 动作：最后一段只保留一个具体判断或一个没完全想透的问题

## 四、模仿时只学骨架，不搬主题

模仿一个作者，不等于照搬他的：

- 主题偏好
- 专有口头禅
- 表情包和梗
- 固定段落格式

真正该学的是：

- 怎么起手
- 怎么推进
- 怎么证明
- 怎么暴露自己的判断
- 怎么停住

## 四点五、改写成“借骨架，不借人格”

这是给你现在这个 skill 最重要的一条。

如果样本作者和当前作者不是一个人，模仿时要强制拆成两部分：

### 可以借的

- 开头怎么把读者带进来
- 文章怎么分段推进
- 问题和问题之间怎么接
- 证据怎么组织
- 什么地方先讲浅层，再讲深层

### 不能借的

- 样本作者的身份感
- 样本作者稳定使用的口头禅
- 样本作者对读者说话的腔调
- 样本作者的情绪强度
- 样本作者没经历过、但你也没经历过的案例

### 一个简单判断法

写完后检查：

- 如果把文中的项目名、经历名词全替换掉，这篇是不是还像你本人？
- 如果只是“像小林在讲你的经历”，那就是借过头了
- 如果是“你的经历，但推进方式更清楚了”，那才是借对了

## 五、最有效的练习法

### 1. 反向提纲

把样本每一段写成一句“这段在干嘛”：

- 建立人物和场景
- 抛一个常见问题
- 用个人经历提高 stakes
- 拿案例把抽象问题落地
- 再一点点推到技术判断

做完之后，你会看到作者真正的推进方式，而不是只看到表面句子。

### 2. 仿写，不抄写

选一段样本，不抄句子，只保留它的推进骨架，用你自己的题材重写。

例如：

- 样本骨架：`读者故事 -> 为什么这个问题值得讲 -> 用一个系统案例把基础知识串起来`
- 你的题材：`自己项目里的一个性能坑 -> 为什么这个坑让我重新看线程模型 -> 用一次真实排查把知识串起来`

仿写时再补一个限制：

- 每一节至少放一个只能由你自己提供的细节
- 如果某一节完全可以脱离你的经历独立成立，那一节通常还太像模板

### 3. 做“去口头禅”测试

把样本作者的明显口头禅删掉，再看这段还像不像他。

如果删掉以后风格就消失了，说明你提取到的只是表面特征。

### 4. 做“换题材”测试

把同样的风格骨架换到另一个主题上，看还成不成立。

如果一换题材就写不动，说明你学到的是内容，不是文风。

## 六、提取结果的推荐格式

建议最终写成四段：

1. `可观察特征`
2. `推测性偏好`
3. `不要盲目模仿的表面特征`
4. `可直接练习的动作`

## 七、如何学习一种文风

如果你真想学，不要停留在“总结出几个特点”。

更有效的顺序是：

1. 选 3 到 5 篇同作者文章
2. 每篇都做反向提纲
3. 找重复出现的推进模式
4. 把这些模式改写成动作卡片
5. 每次写自己的文章时，只强行练 1 到 2 个动作
6. 写完再对照样本，检查自己是学到骨架，还是学成口癖

这比一次性“完全模仿”靠谱得多。

## 八、给当前作者的适配提醒

你当前稳定偏好是：

- 从认知来路切入
- 保留误解、犹豫和未完全想通的部分
- 不喜欢“我来给你讲明白”的教师腔

所以如果要借小林这类样本，推荐只借这些：

- 主问题驱动
- 从简单理解走向更深理解
- 用一个主案例或主线问题串内容
- 每一节都回答一个自然追问

不建议借这些：

- 过强的“面向读者上课”姿态
- 过多的鼓动式喊话
- 过满的确定性
