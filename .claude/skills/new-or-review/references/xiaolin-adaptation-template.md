# 仿小林骨架，但仍然是作者本人

这份模板不是让文章“像小林说话”，而是让文章借到两篇样本里最稳定的推进骨架：

- 低门槛入口
- 主问题驱动
- 每节回答一个自然追问
- 先讲浅层理解，再逐层加深

同时强制保留作者自己的：

- 经历
- 误解
- 证据
- 判断
- 迟疑和没完全想通的部分

## 一、适用场景

适合：

- 技术深潜
- 读书笔记
- 学习方法总结
- 系统设计复盘
- “我是怎么把几个知识点串起来的”这类文章

不太适合：

- 纯情绪随笔
- 纯抽象方法论
- 没有作者本人材料支撑的泛泛科普

## 二、起稿前先收集四类材料

写之前，先把下面四类东西补出来。不够就先访谈，不要硬写。

### 1. 低门槛入口

从下面四种里至少选一种：

- 一个真实困惑
- 一个真实人物
- 一个真实项目场景
- 一个真实误解

例子：

- 我当时为什么会在意这个问题？
- 是谁问了我一句什么话？
- 我在哪次项目里第一次被这个问题卡住？

### 2. 主问题

全文只保留一个主问题，别摊太开。

例子：

- 为什么我后来不再把这件事理解成 X？
- 为什么这个系统问题逼着我重新看操作系统？
- 为什么我现在读技术书不再从头看到尾？

### 3. 逐层加深的三到四步

推荐从下面挑 3 到 4 层：

- 最早的直觉或误解
- 低门槛理解
- 第一次发现这个理解不够用
- 项目或实验里的真实证据
- 现在更能接受的解释
- 还没完全想通的边界

### 4. 只能由作者本人提供的锚点

每一节至少有一个：

- 自己做过的实验
- 自己看过哪本书哪一章
- 自己项目里的日志、数据、命令、代码
- 自己在实习或项目中改过的判断
- 自己到现在还保留的疑问

## 三、提纲模板

直接按下面这个骨架列提纲。

### 标题方向

- 一个真实问题 + 一个具体对象
- 不写太满，不要一上来就像标准答案

例子：

- 我后来是怎么把 XX 串起来的
- 为什么 XX 这件事，我一直到做了 YY 才想明白
- 我现在看 XX，更愿意从这个角度理解

### 结构模板

#### 1. 开头：低门槛入口

目标：

- 先把读者带进来
- 不急着给定义

可以写：

- 一个真实提问
- 一个读者或同学问我的问题
- 我自己最早的误解
- 一次把我卡住的场景

#### 2. 为什么这个问题值得讲

目标：

- 把 stakes 立起来

可以写：

- 它为什么不只是概念问题
- 它为什么会影响项目、学习路径或判断方式
- 我为什么后来越来越在意它

#### 3. 先给低门槛理解

目标：

- 先让读者跟得上

可以写：

- 我最开始是怎么粗糙理解它的
- 这种理解在小场景里为什么看着还成立

#### 4. 低门槛理解在哪里失效

目标：

- 把文章往深处推

可以写：

- 是哪次项目、读书、排查让我发现不对
- 原来的理解到底解释不了什么

#### 5. 用主案例或主问题往下推

目标：

- 不平铺概念，而是顺着一个主线推进

可以写：

- 一个系统案例
- 一次性能问题
- 一个读书路径
- 一次设计决策

这一节之后，每小节都问一次：

- 读者下一步最自然会问什么？

然后下一节直接回答它。

#### 6. 现在我更愿意怎么理解

目标：

- 给出当前判断

要求：

- 判断要从前面的经历和证据里长出来
- 不要像老师宣布标准答案

#### 7. 结尾：停在具体判断或未完问题

可以这样收：

- 我现在更倾向于……
- 这件事我到现在还没完全想通……
- 后来再碰到类似问题时，我会先看……

避免：

- 空泛升华
- 一二三总结大道理
- 强行拔高

## 四、写作提示词骨架

下面这段可以直接作为内部写作约束来用。

```text
请写一篇中文技术文章。目标不是模仿小林的口头禅，而是借他的结构骨架：

1. 开头先用低门槛入口带读者进入，可以是作者自己的真实困惑、真实人物、真实场景或真实误解。
2. 全文围绕一个主问题推进，不按教材目录平铺。
3. 每一节都回答一个自然追问。
4. 先给低门槛理解，再逐层加深。
5. 所有关键判断都必须落回作者自己的经历、证据、项目场景、日志、代码、读书路径或认知转折。
6. 保留作者本人的迟疑、修正和未完全想通的部分。

同时严格避免：

- 写成“小林在讲作者的经历”
- 使用过强的教师腔、鼓动式喊话和固定口头禅
- 只剩结构，没有作者自己的材料
- 一上来就宣布标准答案
- 结尾强行升华

文章里每一大节至少出现一个只能由作者本人提供的细节。
如果某一节脱离作者经历也能独立成立，说明这一节还太空，需要继续落地。
```

## 五、审稿自检清单

写完后至少检查这 10 条：

1. 开头是不是一个低门槛入口，而不是抽象定义？
2. 全文是不是只有一个主问题在带路？
3. 每一节之间是不是都能回答“下一步自然会问什么”？
4. 有没有先给低门槛理解，再逐层加深？
5. 每一节有没有至少一个作者本人才能提供的细节？
6. 当前判断是不是从经历和证据里长出来的？
7. 有没有哪一段像标准教程，而不像作者本人在复盘？
8. 有没有哪一段像“小林在讲课”，而不是作者本人在说？
9. 结尾是不是停在具体判断、后续动作或未完问题上？
10. 把口头禅和喊话删掉后，这篇还像不像作者本人？

## 六、最短执行版

如果以后要在 skill 里快速调用，就记这 5 条：

1. 低门槛入口起手
2. 一个主问题串全文
3. 每节回答一个自然追问
4. 先浅后深
5. 每节必须有作者自己的锚点
