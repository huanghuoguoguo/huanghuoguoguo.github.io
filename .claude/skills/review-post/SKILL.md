# 博客审稿

对指定的博客文章进行审阅，检查以下维度：

## 检查清单
1. **技术准确性**：术语、数据、原理是否正确，有没有过时的说法
2. **论证逻辑**：前提是否支撑结论，有没有为了论点而夸大问题（参考 RAG 文章向量更新的教训）
3. **结构完整性**：开头是否有切入点，结尾是否有反思，中间是否有数据支撑
4. **风格与排版校验（重点去 AIGC 味）**：
   - **请务必在审阅前参考/读取 `.claude/skills/new-post/SKILL.md` 中定义的“语言与排版风格规范”**。
   - 检查文章视角的“人味”：是否有鲜明的第一人称与一线实践视角，是否敢于讨论具体痛点和 Trade-off？
   - 严查并剔除 AI 惯用词：如发现“众所周知”、“随着科技的发展”、“不可否认的是”等陈词滥调，必须指出并要求修改为接地气的大白话或生动比喻。
   - 检查排版呼吸感：段落是否过长，是否善用加粗/短句/列表来辅助跳读。
   - 减少“不是，而是”这类描述，这是最严重的ai味。
   - 非必要不使用“”双引号来包裹内容，非必要不使用：冒号来说明接下来的言论。
5. **frontmatter**：title/date/categories/tags 是否完整

## 输出格式
- 按严重程度分级：❌ 事实错误 / ⚠️ 逻辑问题 / 💡 改进建议
- 每条给出具体位置和修改建议
- 最后给一个整体评价
