# 博客审稿

对指定的博客文章进行审阅，检查以下维度：

## 检查清单
1. **技术准确性**：术语、数据、原理是否正确，有没有过时的说法
2. **论证逻辑**：前提是否支撑结论，有没有为了论点而夸大问题（参考 RAG 文章向量更新的教训）
3. **结构完整性**：开头是否有切入点，结尾是否有反思，中间是否有数据支撑
4. **风格一致性**：是否符合作者一贯的口语化+数据驱动风格
5. **frontmatter**：title/date/categories/tags 是否完整

## 输出格式
- 按严重程度分级：❌ 事实错误 / ⚠️ 逻辑问题 / 💡 改进建议
- 每条给出具体位置和修改建议
- 最后给一个整体评价
