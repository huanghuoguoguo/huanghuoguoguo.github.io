# 信息抽取与回写

## 如何从对话里抽取信息

每次触发时，先把本轮信息分成四类：

- `硬事实`
  - 时间、项目、职责、结果、技术细节、指标

- `一手判断`
  - 用户明确表达的技术看法、选型理由、方法论

- `思考方式`
  - 用户怎么排查问题、怎么读书、怎么形成判断

- `风格偏好`
  - 用户删什么句子，讨厌什么腔调，喜欢什么表达推进方式

如果一句话同时属于两类，可以分别沉淀。

例如：

- “dooba 最大的问题不是编码，是我把需求想大了”
  - `PROFILE.md -> 思考方式`
  - `经历与思考素材库.md -> 一张素材卡片`

- “我不喜欢那种太中性、太客观、像已经审过一遍的稿子”
  - `PROFILE.md -> 风格偏好`

- “曙光那个 GC 问题是 5s 一轮、3 万对象、Young GC 压力太大”
  - `中科曙光经历.md`
  - 如有认知转折，再补进素材库

## 来自写作与改稿任务时，怎么沉淀

这是和 `new-or-review` 联动时最容易写乱的地方。

### 可以作为沉淀依据的信号

- 用户在访谈里明确说出的经历、判断、边界
- 用户对 AI 稿件做出的真实删改
- 用户连续多次表达的稳定偏好
- 用户补充的项目事实、指标、职责边界

### 默认不能直接沉淀的信号

- 检测器给出的 `signals`
- AI 自己推测出来的“你可能偏好……”
- 为了当前文章临时采用的润色策略
- 只在一篇文章里成立、还没确认是稳定偏好的写法

### 落点规则

- 访谈中得到的认知转折、原话、场景：
  - 优先写进 `_not_publish/经历与思考素材库.md`

- 用户在改稿时表现出的稳定风格偏好：
  - 写进 `author-profile/PROFILE.md`

- 写作过程中补出来的项目事实、时间线、职责边界：
  - 写进对应经历文件或 `_not_publish/jianli.md`

- 只对当前一篇文章有效的结构决定：
  - 留在文章或 `new-or-review` 的上下文里，不写进画像

## 实际步骤

1. 读取 `PROFILE.md`
2. 判断本轮对话有没有新信息
3. 如有，按上面的四类分拣
4. 决定落点：
   - 稳定认识 -> `PROFILE.md`
   - 可写成博客的认知转折 -> `经历与思考素材库.md`
   - 事实修正 -> 对应经历文件 / `jianli.md`
5. 检查是否和旧内容冲突
6. 用最小改动更新，不大面积重写
7. 如有更新，运行：
   - `python3 .claude/skills/author-profile/scripts/check_author_profile.py`
8. 如果没有新信息，就不改
