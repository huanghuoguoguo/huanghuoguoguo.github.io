# 新建博客文章

根据用户给出的主题创建新文章。

## 步骤
1. 确认主题、目标读者、文章类型（技术深潜/系统设计/读书笔记/实践复盘）
2. 生成文件：`_posts/YYYY-M-D-标题.md`，日期用当天
3. 填写 frontmatter（title, date, categories, tags）
4. 根据文章类型生成骨架结构，不填充内容
5. 骨架中用注释标注每节应该写什么、大概多少字

## 文章类型模板

### 技术深潜
适用于深入某个技术点的文章（如 eBPF、HNSW、MVCC）。
- 开头：问题或场景引入
- 中间：原理剖析 + 代码/数据验证
- 结尾：适用场景总结 + 元认知反思

### 系统设计
适用于架构设计类文章（如 RAG 系统、日志系统）。
- 开头：面试题/业务场景引入
- 中间：角色划分 → 选型推导 → 关键决策点
- 结尾：与通用方案的差异对比 + 个人经历关联

### 读书笔记
适用于技术书籍的读后整理。
- 开头：为什么读这本书，带着什么问题
- 中间：核心观点提炼 + 自己的理解和批注
- 结尾：对自己实际工作/项目的启发

### 实践复盘
适用于实际项目的踩坑记录（如 db_gen、OB 大赛）。
- 开头：要解决的问题
- 中间：迭代过程（V1→V2→...），每版贴数据、贴瓶颈分析
- 结尾：最终方案 + 反思哪些弯路可以避免
